JavaScript имеет два основных типа данных: **примитивы** и **объекты**

## Примитивные типы

Примитивы — это базовые типы данных. Их значение неизменно (immutable), а сами они передаются **по значению**.
<Callout title="Важно" type="info">
Так сложилось что под капотом примитивы - это тоже объекты. Больше можно прочитать [здесь](#%D0%B1%D0%BE%D0%BB%D0%B5%D0%B5-%D0%B3%D0%BB%D1%83%D0%B1%D0%BE%D0%BA%D0%BE%D0%B5-%D0%BF%D0%BE%D0%B7%D0%BD%D0%B0%D0%BD%D0%B8%D0%B5-%D1%82%D0%B8%D0%BF%D0%BE%D0%B2-%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85-%D0%B2-js)
</Callout>

### Список примитивов

- **Number** — числа (`42`, `3.14`, `NaN`, `Infinity`)
- **String** — строки (`"hello"`, `'world'`, `` `template` ``)
- **Boolean** — логические значения (`true`, `false`)
- **Null** — явное отсутствие значения
- **Undefined** — неопределённое значение
- **Symbol** (ES6) — уникальные идентификаторы
- **BigInt** (ES2020) — большие целые числа



<Callout title="Важно">
 Примитивы неизменяемы. Например, методы для строк не изменяют саму строку, а возвращают новую.
</Callout>

## Объекты

Объекты — это коллекции данных и функциональности. Они передаются **по ссылке**.

### Ключевые особенности непримитивов:
1. Передача по ссылке
```js
let a = { name: "Alice" };
let b = a;
b.name = "Bob";
console.log(a.name); // "Bob" (изменился исходный объект)
```
2. Наличие методов
```js
let arr = [1, 2, 3];
arr.push(4); // метод массива
console.log(arr); // [1, 2, 3, 4]
```
3. Сравнение по ссылке
```js
console.log({} === {}); // false (разные объекты)
console.log([] === []); // false (разные массивы)
```
4. Пример с const
```js
const obj = { x: 10 };
obj.x = 20; // Можно изменять свойства
console.log(obj.x); // 20
// obj = { x: 30 }; // Ошибка - нельзя переприсваивать
```
5. Поверхностное копирование
```js
let original = { a: 1, b: { c: 2 } };
let copy = { ...original };
copy.b.c = 3;
console.log(original.b.c); // 3 (изменилось и в оригинале)
```

### Подтипы объектов (встроенные):
В JavaScript почти все сложные структуры, кроме примитивов, являются объектами. Вот основные подтипы:
#### 1. Обычные объекты (Object)
```js
let obj = { name: "Alice", age: 25 };
```
#### 2. Массивы (Array)
```js
let arr = [1, 2, 3];
console.log(typeof arr); // "object" (но Array.isArray(arr) → true)
```
#### 3. Функции (Function)
```js
function greet() { console.log("Hello!"); }
console.log(typeof greet); // "function" (но это объект)
```
#### 4. Даты (Date)
```js
let now = new Date();
```
#### 5. Регулярные выражения (RegExp)
```js
let regex = /\d+/g;
```
#### 6. Коллекции (ES6+):
- Map – словарь с любыми ключами.
- Set – множество уникальных значений.
- WeakMap и WeakSet – аналоги Map/Set с weak-ссылками.
#### 7. Ошибки (Error и подтипы):
```js
let error = new Error("Oops!");
```
#### 8. Другие встроенные объекты:
- Promise (ES6)
- Proxy (ES6)
- Intl (интернационализация)
- ArrayBuffer и TypedArray (бинарные данные)

### Особенности typeof:
```javascript
typeof 42;          // "number"
typeof "hello";     // "string"
typeof true;        // "boolean"
typeof undefined;   // "undefined"
typeof null;        // "object" (историческая фича в JS)
typeof Symbol();    // "symbol"
typeof 10n;         // "bigint"
typeof {};          // "object"
typeof [];          // "object"
typeof function(){};// "function" 
```

### Как на самом деле хранятся данные в V8?
Ниже приведу **TLDR:**

#### Числа (Number):

        Маленькие целые числа (-(2³⁰-1) до +(2³⁰-1)) хранятся как Smi (31-битные значения)

        Большие числа и числа с плавающей точкой — как HeapNumber (специальные объекты в куче)

#### Строки (String):

        Все строки — объекты (несмотря на "примитивность" в спецификации)

        V8 использует String Pool — одинаковые строки ссылаются на один объект

#### Boolean/Null/Undefined:

        Хранятся как объекты типа Oddball (предопределённые константы)

        true, false, null, undefined создаются один раз при старте движка

#### Главный вывод:

        В V8 почти все типы — объекты "под капотом"

        "Примитивы" в JS — это логическая абстракция, физическая реализация сложнее

        Оптимизации движка (Smi, String Pool, Oddball) ускоряют работу с данными

#### **Ключевая мысль:**
Спецификация JS описывает поведение, а движки (вроде V8) реализуют его оптимальным способом, который может отличаться от "теоретического" представления.
